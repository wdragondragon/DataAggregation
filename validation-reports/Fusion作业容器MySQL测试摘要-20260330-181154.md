# Fusion作业容器MySQL测试摘要-20260330-181154

## 1. 测试范围

本次摘要覆盖 `data-mock` 模块中的 Fusion JobContainer MySQL 测试，包含：

- 基础正确性测试
- 100 / 1w 数据量 benchmark
- 局部乱序吸收测试
- 严重乱序触发 spill 测试
- spill 清理后 `activeSpillBytes` 释放测试

相关测试类：

- `data-mock/src/test/java/com/jdragon/aggregation/datamock/fusion/FusionJobContainerMysqlIntegrationTest.java`
- `data-mock/src/test/java/com/jdragon/aggregation/datamock/fusion/FusionJobContainerMysqlBenchmarkTest.java`
- `data-mock/src/test/java/com/jdragon/aggregation/datamock/fusion/FusionJobContainerMysqlAdaptiveBehaviorTest.java`

## 2. 核心结论

- `JobContainer` 驱动的 Fusion MySQL 链路可稳定运行。
- 100 条数据正确性测试通过，目标表结果与预期一致。
- 1w 条 benchmark 通过，中量数据链路稳定。
- 轻微局部乱序可被 source 侧缓冲吸收，执行引擎仍保持 `sortmerge`。
- 严重乱序会进入 `hybrid` 并触发 spill，最终融合结果仍正确。
- spill 场景结束后 `activeSpillBytes` 回到 `0`，说明 spill 文件清理后的 active quota 释放生效。
- 100w 条高压场景入口已就绪，但本次未实际执行。

## 4. 复现命令

基础正确性与 adaptive 行为联合回归：

```bash
mvn -pl data-mock -am "-Dtest=FusionJobContainerMysqlIntegrationTest,FusionJobContainerMysqlAdaptiveBehaviorTest" test -DfailIfNoTests=false
```

单独执行 adaptive 行为测试：

```bash
mvn -pl data-mock -am "-Dtest=FusionJobContainerMysqlAdaptiveBehaviorTest" test -DfailIfNoTests=false
```

执行 100w 高压场景：

```bash
mvn -pl data-mock -am "-Dtest=FusionJobContainerMysqlBenchmarkTest" test -DfailIfNoTests=false -DrunFusionJobContainerMysqlBenchmarks=true -DfusionJobContainerScenario=fusion_job_large_bench
```

## 5. 当前建议

- 如果目标是验收功能正确性，当前结果已经足够支撑 Fusion MySQL JobContainer 链路通过。
- 如果目标是评估极限压力，下一步建议单独运行 100w 场景。
- 如果目标是验证 spill guard 的失败边界，建议后续引入更大 payload 或更慢释放模型，避免仅靠小 `maxSpillBytesMB` 得到不稳定结果。
