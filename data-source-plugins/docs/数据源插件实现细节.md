# 数据源插件实现细节

## 架构概述

DataAggregation 的数据源插件体系基于 **插件化架构** 设计，通过统一的插件加载框架实现数据连接能力的可扩展性。整个体系分为三个层次：

1. **插件加载框架层**：提供插件发现、加载、管理的核心机制
2. **抽象接口层**：定义三类插件的标准接口和抽象基类
3. **具体实现层**：各个数据源的具体插件实现

## 插件加载框架

### AbstractPlugin - 插件基类

所有插件的根基类，提供插件的基本属性和生命周期管理：

```java
package com.jdragon.aggregation.pluginloader.spi;

import com.jdragon.aggregation.commons.util.Configuration;
import com.jdragon.aggregation.pluginloader.type.IPluginType;

public abstract class AbstractPlugin {
    // 插件配置
    protected Configuration pluginConf;
    // 插件类型
    protected IPluginType pluginType;
    // 类加载器
    protected ClassLoader classLoader;
    
    // Getter/Setter 方法...
    
    // 初始化方法（可被子类覆盖）
    public void init() {}
    
    // 销毁方法（可被子类覆盖）
    public void destroy() {}
}
```

定义插件类型的标识接口：

### LoadUtil - 插件加载工具类

核心的插件加载器，负责根据插件类型和名称加载插件类并实例化：

```java
package com.jdragon.aggregation.pluginloader;

public class LoadUtil {
    // 加载插件的主要方法
    public static <T extends AbstractPlugin> T loadJobPlugin(
            IPluginType pluginType, String pluginName) {
        // 1. 加载插件类

        jobPlugin.setPluginConf(ConfigParser.parseOnePluginConfig(pluginType, pluginName));
        jobPlugin.setPluginType(pluginType);
        jobPlugin.setClassLoader(clazz.getClassLoader());
        
        return (T) jobPlugin;
    }
    
    // 加载插件类的私有方法
    private static synchronized Class<? extends AbstractPlugin> loadPluginClass(
            IPluginType pluginType, String pluginName) {
        // 通过 JarLoader 加载类...
    }
}
```

### PluginClassLoaderCloseable - 类加载器管理

管理插件类加载器的生命周期，确保类加载器正确切换和释放：

```java
package com.jdragon.aggregation.pluginloader;

public final class PluginClassLoaderCloseable implements AutoCloseable {
    private final ClassLoaderSwapper classLoaderSwapper;
    private final IPluginType pluginType;
    private final String pluginName;
    
    // 静态工厂方法
    public static PluginClassLoaderCloseable newCurrentThreadClassLoaderSwapper(
            IPluginType pluginType, String pluginName) {
        return new PluginClassLoaderCloseable(pluginType, pluginName);
    }
    
    // 加载插件实例
    public <T extends AbstractPlugin> T loadPlugin() {
        return LoadUtil.loadJobPlugin(pluginType, pluginName);
    }
    
    // 自动恢复原始类加载器
    @Override
    public void close() {
        classLoaderSwapper.restoreCurrentThreadClassLoader();
    }
}
```

### IDataSourceSql - SQL 模板接口

通过接口隔离不同数据库的 SQL 语法差异：

```java
package com.jdragon.aggregation.datasource;

public interface IDataSourceSql {
    // SQL 模板常量
    String DATA_PREVIEW = "select * from {0} limit {1}";
    String TRUNCATE_TABLE = "truncate table {0}";
    String INSERT_DATA = "insert into {0}({1}) values {2}";
    String MYSQL_QUERY_TABLE_SIZE = "SELECT ROUND((DATA_LENGTH + INDEX_LENGTH) / 1024 / 1024, 2) AS data FROM information_schema.TABLES WHERE table_schema =''{0}'' and table_name = ''{1}''";
    String TABLE_COUNT = "SELECT COUNT(*) AS table_count FROM {0}";
    
    // 默认方法提供通用模板
    default String getDataPreview() { return DATA_PREVIEW; }
    default String getTruncateTable() { return TRUNCATE_TABLE; }
    default String getInsertData() { return INSERT_DATA; }
    default String getTableSize() { return MYSQL_QUERY_TABLE_SIZE; }
    default String getTableCount() { return TABLE_COUNT; }
}
```

```java
        String username = configuration.getNecessaryValue("username", ...);
        // ... 其他参数
        
        // 建立 FTP 连接
        this.ftpClient = new FTPClient();
        ftpClient.connect(host, port);
        ftpClient.login(username, password);
        
        return true;

            ftpClient.logout();
            ftpClient.disconnect();
        }
    }
```

### 消息队列插件实现模式

以 Kafka 插件为例：

```java
package com.jdragon.aggregation.datasource.queue.kafka;

import com.jdragon.aggregation.datasource.queue.QueueAbstract;

public class KafkaQueue extends QueueAbstract {
    private KafkaProducer<String, String> producer;
    private KafkaConsumer<String, String> consumer;
    private String topic;
    
    @Override
    public void init() {
        // 从 pluginQueueConf 解析配置
        this.topic = pluginQueueConf.getString("topic");
        Properties props = new Properties();
        
        // 将所有配置转换为 Properties（除 topic 外）
        for (Map.Entry<String, Object> entry : pluginQueueConf.getMap().entrySet()) {
            if (!"topic".equals(entry.getKey())) {
                props.put(entry.getKey(), entry.getValue());
            }
        }
        
        // 创建生产者和消费者
        this.producer = new KafkaProducer<>(props);
        this.consumer = new KafkaConsumer<>(props);
        consumer.subscribe(Collections.singletonList(topic));
    }
    
    @Override
    public void sendMessage(String message) throws Exception {
        producer.send(new ProducerRecord<>(topic, message));
    }
    
    @Override
    public void receiveMessage(Function<String, Boolean> messageProcessor) throws Exception {
        while (true) {
            ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
            for (ConsumerRecord<String, String> record : records) {
                boolean continueConsume = messageProcessor.apply(record.value());
                if (!continueConsume) {
                    return;
                }
            }
        }
    }
    
    @Override
    public void destroy() {
        if (producer != null) producer.close();
        if (consumer != null) consumer.close();
    }
}
```

### plugin.json - 插件注册文件

```json
  "developer": "jdragon"
```

### ConfigParser - 配置解析器

解析插件配置的工具类：

```java
package com.jdragon.aggregation.pluginloader;

public class ConfigParser {
    // 解析单个插件的配置
    public static Configuration parseOnePluginConfig(
            IPluginType pluginType, String pluginName) {
        // 1. 构建插件配置文件路径
        String pluginPath = StringUtils.join(
            new String[]{SystemConstants.PLUGIN_HOME, 
                        pluginType.getName(), 
                        pluginName, 
                        "plugin.json"}, 
            File.separator);
        
        // 2. 读取并解析 JSON 配置
        String content = FileUtils.readFileToString(new File(pluginPath), "UTF-8");
        JSONObject json = JSONObject.parseObject(content);
        
        // 3. 转换为 Configuration 对象
        Configuration config = Configuration.newDefault();
        config.set(generatePluginKey(pluginType, pluginName), json);
        
        return config;
    }
}
```

### 插件配置加载流程

```
1. LoadUtil.loadJobPlugin(pluginType, pluginName)
   ├── 调用 loadPluginClass(pluginType, pluginName)
   │   ├── 生成 pluginKey: "plugin.source.插件名"
   │   ├── 构建 pluginPath: "PLUGIN_HOME/source/插件名/"
   │   ├── 通过 JarLoader 加载插件类
   │   └── 返回 Class<? extends AbstractPlugin>
   ├── 实例化插件类
   ├── 调用 ConfigParser.parseOnePluginConfig() 解析配置
   ├── 设置插件配置、类型、类加载器
   └── 返回插件实例
```

```java

import com.jdragon.aggregation.datasource.queue.QueueAbstract;

public class 你的消息队列Queue extends QueueAbstract {
    // 消息队列客户端实例
    private 你的客户端 client;
    
    @Override
    public void init() {
        // 从 pluginQueueConf 解析配置
        String server = pluginQueueConf.getString("server");
        // 初始化客户端
        this.client = new 你的客户端(server);

    @Override
    public void sendMessage(String message) throws Exception {

    @Override
    public void receiveMessage(Function<String, Boolean> messageProcessor) throws Exception {
        while (true) {
            String message = client.receive();
            boolean continueConsume = messageProcessor.apply(message);
            if (!continueConsume) {
                break;
            }
        }
    }
    
    @Override
    public void destroy() {
        if (client != null) {
            client.close();
        }
    }
}
```

## 总结

DataAggregation 的数据源插件体系设计精良，通过分层架构和标准化接口实现了高度的可扩展性：

1. **统一加载机制**：基于 `plugin.json` 和 `LoadUtil` 的插件发现与加载
2. **标准化接口**：三类插件都有明确的接口定义和抽象基类
3. **配置驱动**：通过 DTO 或 Configuration 对象统一传递配置
4. **资源管理**：支持 AutoCloseable 和生命周期管理
5. **错误处理**：统一的异常体系和错误码

开发新插件时，只需遵循相应的接口规范，实现必要的业务逻辑，即可无缝集成到 DataAggregation 的数据处理流程中。这种设计既保证了框架的稳定性，又为业务扩展提供了充分的灵活性。
