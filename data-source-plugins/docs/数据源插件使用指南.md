# 数据源插件使用指南

## 概述

DataAggregation 框架通过 `data-source-plugins` 模块提供了丰富的插件化数据连接能力，支持 **数据源（数据库/数据仓库）**、**文件源（文件系统/对象存储）** 和 **消息队列** 三大类数据接入方式。所有插件遵循统一的插件加载机制，通过简单的配置即可快速集成到数据同步、ETL 等任务中。

## 插件加载机制

### PluginClassLoaderCloseable 与插件加载

插件通过 `com.jdragon.aggregation.pluginloader.PluginClassLoaderCloseable` 进行加载，采用 `try-with-resources` 语法确保类加载器正确释放：

```java
try (PluginClassLoaderCloseable loaderSwapper =
         PluginClassLoaderCloseable.newCurrentThreadClassLoaderSwapper(SourcePluginType.SOURCE, "插件名称")) {
    // 加载插件实例
    AbstractDataSourcePlugin sourcePlugin = loaderSwapper.loadPlugin();
    // 或：FileHelper fileHelper = loaderSwapper.loadPlugin();
    // 或：QueueAbstract queue = loaderSwapper.loadPlugin();
    
    // 使用插件...
}
```

### plugin.json 注册文件

每个插件模块的 `src/main/resources/plugin.json` 是插件的注册描述符，用于 Java SPI 机制下的插件发现。格式如下：

```json
{
  "name": "插件标识符",
  "class": "插件实现类的全限定名",
  "description": "插件描述",
  "developer": "开发者"
}
```

插件加载器会根据 `plugin.json` 中的 `class` 字段动态实例化插件类。

## 配置参数详解

### 数据源插件配置（BaseDataSourceDTO）

所有数据源插件均通过 `com.jdragon.aggregation.datasource.BaseDataSourceDTO` 对象传递配置：

| 字段 | 类型 | 必填 | 默认值 | 描述 |
|------|------|------|--------|------|
| `name` | String | 否 | - | 数据源名称（用于标识） |
| `type` | String | 否 | - | 数据源类型（插件内部自动设置） |
| `host` | String | **是**（大多数） | - | 主机 IP 地址或服务端点 |
| `port` | String | **是**（大多数） | - | 端口号 |
| `database` | String | **是**（大多数） | - | 数据库名/项目名/存储桶名 |
| `userName` | String | **是**（大多数） | - | 用户名 |
| `password` | String | **是**（大多数） | - | 密码 |
| `other` | String | 否 | - | 数据库连接参数字符串，如 `useSSL=false,serverTimezone=UTC` |
| `usePool` | boolean | 否 | `false` | 是否使用连接池 |
| `extraParams` | Map<String, String> | 否 | `new HashMap<>()` | 扩展参数映射，支持数据库特定参数 |
| `bucket` | String | 否 | - | 主要用于 InfluxDB，类似 schema |
| `principal` | String | 否 | - | Kerberos principal（Hive/HDFS 认证） |
| `keytabPath` | String | 否 | - | Kerberos keytab 文件路径 |
| `krb5File` | String | 否 | - | krb5 配置文件路径 |
| `jdbcUrl` | String | 否 | - | JDBC URL（通常由插件自动生成） |
| `driverClassName` | String | 否 | - | 驱动类名（插件内部设置） |

**特有插件注意事项**：
- **OdpsSourcePlugin**：`host` 作为 ODPS endpoint，`database` 作为 project，`extraParams` 作为全局设置传递给 ODPS
- **InfluxDBV1SourcePlugin**：`host` 为完整 URL（如 `http://localhost:8086`），`database` 为数据库名
- **InfluxDBSourcePlugin**：`host` 为 URL，`password` 为 token，`database` 为 org，`bucket` 为存储桶名
- **DMSourcePlugin**：`other` 参数中的 `socketTimeout` 和 `connectTimeout` 会设置为系统属性
- **Hive2SourcePlugin/Hive3SourcePlugin**：需要 Kerberos 认证时需提供 `principal`、`keytabPath`、`krb5File`

### 文件源插件配置（Configuration 键值对）

所有文件源插件均通过 `com.jdragon.aggregation.commons.util.Configuration` 对象（键值映射）传递配置：

#### FtpHelper (`ftp`) 配置

| 配置键 | 类型 | 必填 | 默认值 | 描述 |
|--------|------|------|--------|------|
| `host` | String | **是** | - | FTP 服务器地址 |
| `username` | String | **是** | - | 用户名 |
| `password` | String | **是** | - | 密码 |
| `port` | int | 否 | `21` | 端口号 |
| `timeout` | int | 否 | `60000` | 连接超时（毫秒） |
| `connectMode` | String | 否 | `"PASV"` | 连接模式：`"PASV"`（被动模式）或 `"PORT"`（主动模式） |
| `ftpTLS` | String | 否 | `"none"` | TLS 模式：`"none"`（普通 FTP）、`"implicit"`（隐式 FTPS）、`"explicit"`（显式 FTPS） |

#### SftpHelper (`sftp`) 配置

| 配置键 | 类型 | 必填 | 默认值 | 描述 |
|--------|------|------|--------|------|
| `host` | String | **是** | - | SFTP 服务器地址 |
| `username` | String | **是** | - | 用户名 |
| `password` | String | **是** | - | 密码 |
| `port` | int | 否 | `21` | 端口号（默认同 FTP） |
| `timeout` | int | 否 | `60000` | 连接超时（毫秒） |

#### MinioHelper / AliyunOssHelper (`minio`) 配置

| 配置键 | 类型 | 必填 | 默认值 | 描述 |
|--------|------|------|--------|------|
| `endpoint` | String | **是** | - | 对象存储服务端点 |
| `accessKey` | String | **是** | - | 访问密钥 ID |
| `secretKey` | String | **是** | - | 访问密钥 Secret |
| `bucket` | String | **是** | - | 存储桶名称 |

#### HdfsHelper (`tbds-hdfs`, `tbds-hdfs3`) 配置

| 配置键 | 类型 | 必填 | 默认值 | 描述 |
|--------|------|------|--------|------|
| `hdfsSiteFilePath` | String | 否 | - | `hdfs-site.xml` 文件路径或 HTTP URL |
| `coreSiteFilePath` | String | 否 | - | `core-site.xml` 文件路径或 HTTP URL |
| `hadoopConfig` | Map<String, String> | 否 | - | 覆盖 Hadoop 配置的键值对 |
| `kerberosKeytabFilePath` | String | 否 | - | Kerberos keytab 文件路径 |
| `kerberosPrincipal` | String | 否 | - | Kerberos principal |
| `krb5Conf` | String | 否 | - | `krb5.conf` 文件路径 |
| `haveKerberos` | String | 否 | - | 是否启用 Kerberos 认证 |

**注意**：Hadoop 配置可通过 `hadoopConfig` 映射传递，其中必须包含 `hadoop.security.authentication` 等配置键。

### 消息队列插件配置（pluginQueueConf）

所有消息队列插件均通过 `pluginQueueConf`（`Configuration` 对象）传递配置：

#### KafkaQueue (`kafka`) 配置

| 配置键 | 类型 | 必填 | 默认值 | 描述 |
|--------|------|------|--------|------|
| `topic` | String | **是** | - | Kafka 主题名称 |
| `bootstrap.servers` | String | **是** | - | Kafka 集群地址（如 `localhost:9092`） |
| `group.id` | String | 否 | - | 消费者组 ID |
| 其他 Kafka 客户端属性 | 任意 | 否 | - | 所有 `ProducerConfig` / `ConsumerConfig` 支持的属性 |

**注意**：除了 `topic` 被特殊提取外，其余所有配置键都会直接传递给 `Properties` 对象用于创建 Kafka 客户端。

#### RabbitQueue (`rabbitmq`) 配置

| 配置键 | 类型 | 必填 | 默认值 | 描述 |
|--------|------|------|--------|------|
| `queueName` | String | **是** | - | 队列名称 |
| `host` | String | 否 | `"localhost"` | RabbitMQ 服务器地址 |
| `username` | String | 否 | `"guest"` | 用户名 |
| `password` | String | 否 | `"guest"` | 密码 |
| `port` | int | 否 | `5672` | 端口号 |

#### RocketQueue (`rocketmq`) 配置

| 配置键 | 类型 | 必填 | 默认值 | 描述 |
|--------|------|------|--------|------|
| `namesrvAddr` | String | **是** | - | NameServer 地址 |
| `producerGroup` | String | **是** | - | 生产者组名 |
| `accessKey` | String | 否 | - | ACL 访问密钥（可选） |
| `secretKey` | String | 否 | - | ACL 密钥（可选） |
| `topic` | String | **是** | - | 主题名称 |
| `tag` | String | 否 | - | 消息标签（过滤用） |
| `consumerGroup` | String | **是** | - | 消费者组名 |
| `pullBatchSize` | int | 否 | `100` | 批量拉取大小 |
| `pullInterval` | long | 否 | `-1` | 拉取间隔（毫秒，-1 表示不等待） |

## 使用示例

### 数据源插件示例（MySQL）

```java
import com.jdragon.aggregation.datasource.AbstractDataSourcePlugin;
import com.jdragon.aggregation.datasource.BaseDataSourceDTO;
import com.jdragon.aggregation.datasource.SourcePluginType;
import com.jdragon.aggregation.pluginloader.PluginClassLoaderCloseable;

public class MySQLExample {
    public static void main(String[] args) {
        // 1. 准备数据源配置
        BaseDataSourceDTO sourceDTO = new BaseDataSourceDTO();
        sourceDTO.setHost("localhost");
        sourceDTO.setPort("3306");
        sourceDTO.setDatabase("testdb");
        sourceDTO.setUserName("root");
        sourceDTO.setPassword("123456");
        sourceDTO.setOther("useSSL=false&serverTimezone=UTC");
        sourceDTO.setType("mysql8");
        
        // 2. 加载插件并执行操作
        try (PluginClassLoaderCloseable loaderSwapper =
                PluginClassLoaderCloseable.newCurrentThreadClassLoaderSwapper(
                    SourcePluginType.SOURCE, "mysql8")) {
            
            AbstractDataSourcePlugin sourcePlugin = loaderSwapper.loadPlugin();
            
            // 连接测试
            boolean connected = sourcePlugin.connectTest(sourceDTO);
            System.out.println("连接测试: " + (connected ? "成功" : "失败"));
            
            // 获取所有表名
            java.util.List<String> tables = sourcePlugin.getTableNames(sourceDTO, "");
            System.out.println("数据库表列表: " + tables);
            
            // 预览表数据（前10行）
            if (!tables.isEmpty()) {
                String firstTable = tables.get(0);
                com.jdragon.aggregation.commons.pagination.Table<java.util.Map<String, Object>> preview = 
                    sourcePlugin.dataModelPreview(sourceDTO, firstTable, "10");
                System.out.println("表 " + firstTable + " 预览数据: " + preview);
            }
        }
    }
}
```

### 文件源插件示例（FTP）

```java
import com.jdragon.aggregation.commons.util.Configuration;
import com.jdragon.aggregation.datasource.SourcePluginType;
import com.jdragon.aggregation.datasource.file.FileHelper;
import com.jdragon.aggregation.pluginloader.PluginClassLoaderCloseable;
import org.apache.commons.io.IOUtils;

public class FTPExample {
    public static void main(String[] args) {
        try (PluginClassLoaderCloseable loaderSwapper =
                PluginClassLoaderCloseable.newCurrentThreadClassLoaderSwapper(
                    SourcePluginType.SOURCE, "ftp")) {
            
            // 1. 加载插件
            FileHelper fileHelper = loaderSwapper.loadPlugin();
            
            // 2. 准备配置
            Configuration configuration = Configuration.newDefault();
            configuration.set("host", "ftp.example.com");
            configuration.set("port", "21");
            configuration.set("username", "user");
            configuration.set("password", "password");
            configuration.set("timeout", "60000");
            configuration.set("connectMode", "PASV");
            configuration.set("ftpTLS", "none");
            
            // 3. 连接服务器
            boolean connected = fileHelper.connect(configuration);
            if (!connected) {
                throw new RuntimeException("FTP 连接失败");
            }
            
            // 4. 列出目录文件
            java.util.Set<String> files = fileHelper.listFile("/home/user/data", ".*\\.csv");
            System.out.println("CSV 文件列表: " + files);
            
            // 5. 读取文件内容
            if (!files.isEmpty()) {
                String firstFile = files.iterator().next();
                java.io.InputStream inputStream = fileHelper.getInputStream("/home/user/data", firstFile);
                String content = IOUtils.toString(inputStream, java.nio.charset.StandardCharsets.UTF_8);
                inputStream.close();
                System.out.println("文件内容前100字符: " + content.substring(0, Math.min(100, content.length())));
            }
            
            // 6. 写入新文件
            fileHelper.mkdir("/home/user/data/processed");
            java.io.OutputStream outputStream = fileHelper.getOutputStream(
                "/home/user/data/processed", "result.txt");
            IOUtils.write("数据处理完成", outputStream, java.nio.charset.StandardCharsets.UTF_8);
            outputStream.close();
            
            // 7. 关闭连接
            fileHelper.close();
            
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

### 消息队列插件示例（Kafka）

```java
import com.jdragon.aggregation.commons.util.Configuration;
import com.jdragon.aggregation.datasource.SourcePluginType;
import com.jdragon.aggregation.datasource.queue.QueueAbstract;
import com.jdragon.aggregation.pluginloader.PluginClassLoaderCloseable;

public class KafkaExample {
    public static void main(String[] args) {
        try (PluginClassLoaderCloseable loaderSwapper =
                PluginClassLoaderCloseable.newCurrentThreadClassLoaderSwapper(
                    SourcePluginType.SOURCE, "kafka")) {
            
            // 1. 加载插件
            QueueAbstract queue = loaderSwapper.loadPlugin();
            
            // 2. 准备配置
            Configuration queueConf = Configuration.newDefault();
            queueConf.set("topic", "test-topic");
            queueConf.set("bootstrap.servers", "localhost:9092");
            queueConf.set("group.id", "test-group");
            queueConf.set("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
            queueConf.set("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
            queueConf.set("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
            queueConf.set("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");
            queueConf.set("auto.offset.reset", "earliest");
            
            // 3. 设置配置并初始化
            queue.setPluginQueueConf(queueConf);
            queue.init(); // 初始化客户端
            
            // 4. 发送消息
            for (int i = 0; i < 10; i++) {
                String message = "消息-" + i + "-" + java.time.LocalDateTime.now();
                queue.sendMessage(message);
                System.out.println("已发送: " + message);
                Thread.sleep(100);
            }
            
            // 5. 接收消息
            System.out.println("开始接收消息...");
            queue.receiveMessage(message -> {
                System.out.println("收到消息: " + message);
                // 返回 true 继续消费，false 停止消费
                return true;
            });
            
            // 6. 停止消费（示例中不会执行到这里，实际需根据业务控制）
            Thread.sleep(5000);
            
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

## 常见问题

### 1. 如何选择合适的插件类型？
- **关系型数据库**（MySQL、Oracle、PostgreSQL、DM）：使用对应的数据源插件
- **数据仓库**（Hive、ODPS）：使用对应的数据源插件
- **时序数据库**（InfluxDB）：使用 `influxdbv1` 或 `influxdb` 插件
- **文件服务器**（FTP、SFTP）：使用对应的文件源插件
- **对象存储**（MinIO、S3、阿里云 OSS）：使用 `minio` 插件
- **分布式文件系统**（HDFS）：使用 `tbds-hdfs` 或 `tbds-hdfs3` 插件
- **消息队列**：根据使用的中间件选择对应的消息队列插件

### 2. 插件加载失败怎么办？
- 检查 `plugin.json` 文件是否存在且格式正确
- 确认插件模块已正确打包并部署到 `plugin/source/插件名称/` 目录下
- 确认插件依赖的 JAR 包已正确包含
- 检查类路径设置，确保插件类加载器能找到插件类

### 3. 连接测试失败的可能原因？
- **网络连接**：确认主机、端口可访问
- **认证信息**：检查用户名、密码是否正确
- **防火墙**：确认防火墙未阻止连接
- **数据库权限**：确认用户有足够的数据库权限
- **Kerberos 认证**：Hive/HDFS 插件需正确配置 Kerberos 相关参数

### 4. 如何传递数据库连接参数？
- 使用 `BaseDataSourceDTO.setOther()` 方法传递连接参数字符串，如 `useSSL=false&serverTimezone=UTC`
- 对于特定的数据库参数，可通过 `extraParams` Map 传递
- 对于 JDBC URL 中的参数，部分插件会自动拼接，部分需要手动设置 `jdbcUrl`

### 5. Kerberos 认证如何配置？
对于 Hive、HDFS 等支持 Kerberos 认证的数据源：
```java
BaseDataSourceDTO dto = new BaseDataSourceDTO();
// ... 设置基本连接参数
dto.setPrincipal("hive/hostname@REALM");
dto.setKeytabPath("/path/to/keytab/file.keytab");
dto.setKrb5File("/path/to/krb5.conf");
```

### 6. 如何扩展自定义插件？
参考 [数据源插件实现细节](./数据源插件实现细节.md) 文档了解插件开发规范。

## 总结

DataAggregation 的数据源插件框架提供了标准化、插件化的数据连接方案。通过统一的接口设计和配置模式，用户可以轻松地集成各种数据源、文件系统和消息队列到数据处理流程中。掌握插件配置参数和使用方法，能够有效提升数据集成和处理的效率。

对于插件开发和扩展，请参考配套的开发者文档。
